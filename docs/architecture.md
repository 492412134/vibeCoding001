- **可扩展性**：支持水平扩展和模块插拔

### 2.2 核心组件说明

#### 2.2.1 PaymentController
- **职责**：接收HTTP支付请求，参数校验，返回响应
- **关键接口**：
  - `POST /api/payment/pay` - 单笔支付
  - `GET /api/payment/status` - 获取系统状态
  - `GET /api/payment/threads` - 获取线程池状态
  - `POST /api/payment/reset` - 重置指标

#### 2.2.2 PaymentAggregator
- **职责**：请求聚合、队列管理、批次处理、线程池调度
- **核心机制**：
  - **双队列**：VIP队列（PriorityBlockingQueue）和普通队列（LinkedBlockingQueue）
  - **优先级策略**：始终优先处理VIP队列，VIP队列为空时才处理普通队列
  - **批次聚合**：每10条或每1秒触发一次批次处理
  - **线程池**：ThreadPoolExecutor，20核心线程，最大40线程，队列容量1000

#### 2.2.3 ThirdPartyPaymentService
- **职责**：模拟第三方支付接口调用
- **延迟模拟**：50-200ms随机延迟，模拟真实网络延迟
- **成功率**：95%成功率，5%随机失败

### 2.3 数据流转
```
1. 接收请求 → 2. 根据orderType入队 → 3. 批次收集 → 4. 线程池处理
    │              │                      │              │
    │              │                      │              └─ 调用第三方接口
    │              │                      │
    │              │                      └─ 10条或1秒触发
    │              │
    │              └─ VIP → vipQueue
    │                 Normal → normalQueue
    │
    └─ 参数校验 → 生成requestId → 保存PENDING状态

5. 处理结果 → 6. 更新数据库 → 7. 记录指标
    │              │              │
    │              │              └─ QPS/耗时/成功率
    │              │
    │              └─ 更新为SUCCESS/FAILED/TIMEOUT
    │
    └─ 第三方返回结果
```

### 3.2 核心组件说明

#### 3.2.1 条件表达式系统
```java
// 条件接口
interface Condition {
    boolean evaluate(Map<String, Object> context);
}

// 原子条件
class AtomicCondition implements Condition {
    String field;      // 字段名
    String operator;   // 操作符: >, <, ==, >=, <=, !=, contains

// 组合条件
class CompositeCondition implements Condition {

    List<Condition> conditions;

```

## 4. 线程池设计

### 4.1 线程池配置
```java
ThreadPoolExecutor batchProcessor = new ThreadPoolExecutor(
    coreThreads,           // 核心线程数: max(20, CPU核数*2)
    maxThreads,            // 最大线程数: max(40, CPU核数*4)
    60L, TimeUnit.SECONDS, // 空闲线程存活时间
    new LinkedBlockingQueue<>(1000),  // 任务队列容量
    threadFactory,         // 线程工厂（命名线程）
    new CallerRunsPolicy() // 拒绝策略：调用者执行
);
```

### 4.2 线程状态监控
```java
class ThreadStatus {
    String threadName;     // 线程名称
    int processingCount;   // 当前处理元素数量
    boolean isActive;      // 是否正在处理
    long startTime;        // 开始处理时间
    String batchId;        // 当前批次ID
}
```

### 4.3 线程执行流程
```
submit(task)
    │
    ├─ 当前运行线程 < coreThreads ──→ 创建新线程执行任务
    │
    ├─ 当前运行线程 ≥ coreThreads ──→ 任务进入LinkedBlockingQueue
    │                                    │
    │                                    ├─ 队列未满 → 等待执行
    │                                    │
    │                                    └─ 队列已满 → 创建临时线程(maxThreads内)
    │
    └─ 运行线程 = maxThreads 且队列满 ──→ CallerRunsPolicy（调用者执行）
```

## 9. 容错设计

### 9.1 异常处理
```
第三方调用失败
    │
    ├─ 网络超时 ──→ 标记TIMEOUT，可重试
    │
    ├─ 返回失败 ──→ 标记FAILED，记录原因
    │
    └─ 系统异常 ──→ 标记FAILED，告警通知
```

### 9.2 服务恢复
- 重启后自动恢复PENDING状态订单
- 定时任务清理超时订单
- 数据库事务保证数据一致性
