# VS Code 调试配置使用指南

## 什么是 launch.json？

`launch.json` 是 VS Code 的调试配置文件，它告诉 VS Code：
- 如何启动你的程序
- 使用什么参数
- 在哪里设置断点

简单来说，它让你可以**一键启动**程序，不需要手动输入命令。

---

## 如何使用？

### 方法1：使用快捷键（推荐）

1. **打开调试面板**
   - 按 `Ctrl+Shift+D` (Mac: `Cmd+Shift+D`)
   - 或者点击左侧的 **虫子图标** 🐛

2. **选择启动配置**
   - 看顶部有个下拉框（写着"调试"或配置名称）
   - 点击下拉框，选择你想启动的配置

3. **开始调试**
   - 按 `F5`
   - 或者点击绿色的 **"开始调试"** 按钮（三角形图标）

### 方法2：使用命令面板

1. 按 `Ctrl+Shift+P` (Mac: `Cmd+Shift+P`)
2. 输入 `debug`，选择 **"调试：开始调试"**
3. 选择你想启动的配置

## 可用的启动配置

### 一键启动（最常用）

| 配置名称 | 说明 | 启动的服务 |
|---------|------|-----------|
| 🚀 一键启动完整集群（推荐） | 启动所有服务，双节点 | Gateway + Web + Payment×2 + Order×2 |
| 🔧 一键启动核心服务 | 只启动核心服务 | Gateway + Web + Payment×2 |
| 🧪 一键启动（单节点测试） | 每个服务一个节点 | Gateway + Web + Payment + Order |

### 单独启动

| 配置名称 | 说明 | 端口 |
|---------|------|------|
| 🚪 Gateway (端口 8081) | API 网关 | 8081 |
| 🌐 Web 前端 (端口 10990) | 前端页面服务 | 10990 |
| 💰 Payment Service 节点1 | 支付服务第1个节点 | 10991 |
| 💰 Payment Service 节点2 | 支付服务第2个节点 | 10993 |
| 📋 Order Service 节点1 | 订单服务第1个节点 | 10992 |
| 📋 Order Service 节点2 | 订单服务第2个节点 | 10994 |

## 使用示例

### 场景1：第一次启动项目

1. 确保 Nacos 已启动（Docker）
2. 按 `Ctrl+Shift+D` 打开调试面板
3. 顶部下拉框选择 **"🚀 一键启动完整集群（推荐）"**
4. 按 `F5`
5. 等待所有服务启动（会打开多个终端窗口）
6. 访问 http://localhost:8081

### 场景2：只调试支付服务

1. 按 `Ctrl+Shift+D` 打开调试面板
2. 选择 **"💰 Payment Service 节点1"**
3. 按 `F5`
4. 在代码中设置断点（点击行号左侧）
5. 访问 http://localhost:10991 触发断点

### 场景3：测试负载均衡

1. 选择 **"⚡ 仅启动支付服务（双节点）"**
2. 按 `F5`
3. 两个支付服务会同时启动
4. 通过 Gateway 访问，观察请求分发到不同节点

## 调试技巧

### 设置断点

1. 打开任意 Java 文件
2. 点击行号左侧的空白处
3. 出现红点表示断点设置成功
4. 程序运行到这里会暂停

### 查看变量

- 程序暂停时，左侧会显示当前变量的值
- 鼠标悬停在变量上也可以查看

### 单步执行

| 按钮/快捷键 | 功能 |
|------------|------|
| F10 | 单步跳过（不进入方法内部） |
| F11 | 单步进入（进入方法内部） |
| Shift+F11 | 单步跳出（从方法返回） |
| F5 | 继续运行到下一个断点 |
| Shift+F5 | 停止调试 |

### 查看控制台输出

- 每个服务会在独立的终端窗口中运行
- 点击 VS Code 底部的 **"终端"** 标签可以切换
- 或者按 `` Ctrl+` `` (反引号) 打开/关闭终端面板

## 常见问题

### Q: 启动时报错 "端口被占用"

**原因**：之前启动的服务没有正确关闭

**解决**：
1. 在终端中运行 `lsof -i :端口号` 查看占用进程
2. 运行 `kill -9 进程ID` 结束进程
3. 或者重启 VS Code

### Q: 如何停止所有服务？

**方法1**：按 `Shift+F5` 停止当前调试

**方法2**：点击调试工具栏的红色方块按钮

**方法3**：一键启动配置设置了 `"stopAll": true`，停止时会同时停止所有相关服务

### Q: 为什么启动了多个终端窗口？

**答**：因为同时启动了多个服务，每个服务需要一个独立的终端来显示日志

### Q: 如何只查看某个服务的日志？

**答**：点击 VS Code 底部终端面板的下拉框，选择对应的服务终端

## 配置说明

### 单服务配置字段说明

```json
{
    "type": "java",                    // 调试类型：Java
    "name": "显示的名称",               // 在下拉框中显示的名字
    "request": "launch",               // 请求类型：启动
    "mainClass": "主类全路径",          // 程序的入口类
    "projectName": "项目名称",          // Maven 项目名
    "vmArgs": "虚拟机参数",              // JVM 参数，如内存设置、端口等
    "console": "integratedTerminal"     // 使用 VS Code 内置终端
}
```

### 一键启动配置字段说明

```json
{
    "name": "显示的名称",
    "configurations": [                // 要启动的服务列表
        "服务1名称",
        "服务2名称"
    ],
    "stopAll": true                    // 停止时是否同时停止所有服务
}
```

## 最佳实践

1. **开发时**：使用 **"🔧 一键启动核心服务"**，只启动必要的服务
2. **测试负载均衡**：使用 **"⚡ 仅启动支付服务（双节点）"**
3. **完整测试**：使用 **"🚀 一键启动完整集群"**
4. **调试代码**：单独启动需要调试的服务
