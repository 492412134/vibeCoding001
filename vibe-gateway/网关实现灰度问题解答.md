1. 自定义负载均衡器的核心思路

获取灰度规则：从你已有的 GrayRuleConfig 中获取当前服务的灰度策略。
根据灰度规则筛选实例：从 ServiceInstance 列表中过滤出符合要求的实例（例如 metadata 中 version 匹配）。
结合权重/条件决定灰度版本：如果当前请求匹配灰度规则，则返回灰度版本实例；否则返回稳定版本实例。
将自定义负载均衡器注册为 Spring Bean，并使其优先级高于默认的负载均衡器。
2. 实现自定义 ReactiveLoadBalancer

java
@Component
@ConditionalOnMissingBean // 确保不覆盖其他可能的自定义负载均衡器
public class GrayReactiveLoadBalancer implements ReactiveLoadBalancer<ServiceInstance> {

    private final GrayRuleConfig grayRuleConfig;
    private final LoadBalancerClientFactory clientFactory;

    public GrayReactiveLoadBalancer(GrayRuleConfig grayRuleConfig,
                                    LoadBalancerClientFactory clientFactory) {
        this.grayRuleConfig = grayRuleConfig;
        this.clientFactory = clientFactory;
    }

    @Override
    public Mono<Response<ServiceInstance>> choose(Request request) {
        // 1. 获取当前请求的服务名（从 Request 中解析）
        String serviceId = getServiceId(request);
        if (serviceId == null) {
            return Mono.empty();
        }

        // 2. 获取该服务的所有可用实例（通过 LoadBalancerClientFactory 获取）
        ServiceInstanceListSupplier supplier = clientFactory.getInstance(serviceId, ServiceInstanceListSupplier.class);
        return supplier.get(request).next()
                .flatMap(instances -> {
                    // 3. 根据灰度规则选择实例
                    List<ServiceInstance> grayInstances = selectGrayInstances(instances, serviceId);
                    if (grayInstances.isEmpty()) {
                        // 没有灰度实例，回退到默认负载均衡
                        return defaultChoose(instances, request);
                    }
                    // 4. 从灰度实例中按负载均衡策略选一个（可复用默认策略）
                    return chooseFromGrayInstances(grayInstances, request);
                });
    }

    private List<ServiceInstance> selectGrayInstances(List<ServiceInstance> instances, String serviceId) {
        GrayRuleConfig.GrayRule rule = findGrayRule(serviceId);
        if (rule == null || !rule.isEnabled()) {
            // 无灰度规则或未启用，返回空列表，表示使用默认
            return Collections.emptyList();
        }

        // 根据灰度条件（用户ID、IP等）判断当前请求是否应该走灰度
        if (!shouldUseGrayVersion(rule)) {
            return Collections.emptyList();
        }

        // 筛选出版本为 rule.version 的实例
        return instances.stream()
                .filter(instance -> rule.getVersion().equals(instance.getMetadata().get("version")))
                .collect(Collectors.toList());
    }

    private boolean shouldUseGrayVersion(GrayRuleConfig.GrayRule rule) {
        // 这里需要获取当前请求上下文（从 ServerWebExchange 获取）
        // 可以通过 Request 的 attribute 传递灰度标记，或者通过 Context 中的请求头判断
        // 为了演示，简单示例：如果有灰度标记属性，则使用灰度
        return true; // 实际根据你的灰度条件判断
    }

    private Mono<Response<ServiceInstance>> defaultChoose(List<ServiceInstance> instances, Request request) {
        // 回退到默认的负载均衡器（如轮询）
        // 这里可以使用 LoadBalancerClientFactory 的默认实现
        return Mono.just(new DefaultResponse(instances.get(0))); // 简化，实际应使用负载均衡策略
    }

    private Mono<Response<ServiceInstance>> chooseFromGrayInstances(List<ServiceInstance> instances, Request request) {
        // 在灰度实例中按默认策略选一个
        return Mono.just(new DefaultResponse(instances.get(0))); // 同样可改为轮询
    }

    private String getServiceId(Request request) {
        // Request 可能是 ServerWebExchange 的包装，实际中可以通过上下文获取
        // 如果使用的是 LoadBalancerClientFactory 的默认实现，这里可以忽略，由调用方传递
        return null;
    }

    private GrayRuleConfig.GrayRule findGrayRule(String serviceId) {
        if (grayRuleConfig.getRules() == null) return null;
        return grayRuleConfig.getRules().stream()
                .filter(r -> r.getService().equals(serviceId))
                .findFirst()
                .orElse(null);
    }

    // 辅助类：简单响应
    private static class DefaultResponse implements Response<ServiceInstance> {
        private final ServiceInstance instance;
        DefaultResponse(ServiceInstance instance) { this.instance = instance; }
        @Override public ServiceInstance getServer() { return instance; }
        @Override public boolean hasServer() { return instance != null; }
    }
}

方式一：全局替换

方式二：针对特定服务（推荐）
Spring Cloud LoadBalancer 支持为不同服务指定不同的负载均衡器。你可以通过 LoadBalancerClientFactory 的配置来注册服务级别的负载均衡器。

更简单的方式：保持默认负载均衡器不变，但将灰度逻辑封装在 ServiceInstanceListSupplier 中，这样负载均衡器在获取实例列表时就已经被过滤了。

4. 更简单的方案：自定义 ServiceInstanceListSupplier

Spring Cloud LoadBalancer 的核心是 ServiceInstanceListSupplier，它负责提供实例列表。我们可以在获取列表时就根据灰度规则过滤，这样任何负载均衡器（包括默认的轮询）都只会拿到符合灰度要求的实例。

java
@Component
public class GrayServiceInstanceListSupplier implements ServiceInstanceListSupplier {

    private final ServiceInstanceListSupplier delegate;
    private final GrayRuleConfig grayRuleConfig;
    private final ObjectProvider<LoadBalancerProperties> properties;

    public GrayServiceInstanceListSupplier(ServiceInstanceListSupplier delegate,
                                           GrayRuleConfig grayRuleConfig,
                                           ObjectProvider<LoadBalancerProperties> properties) {
        this.delegate = delegate;
        this.grayRuleConfig = grayRuleConfig;
        this.properties = properties;
    }

    @Override
    public String getServiceId() {
        return delegate.getServiceId();
    }

    @Override
    public Flux<List<ServiceInstance>> get() {
        return delegate.get().map(instances -> {
            // 根据当前请求上下文（需要从 Context 获取）决定是否使用灰度实例
            if (shouldUseGray()) {
                // 筛选出灰度版本实例
                return instances.stream()
                        .filter(instance -> instance.getMetadata().getOrDefault("version", "v1").equals("v2"))
                        .collect(Collectors.toList());
            }
            // 否则返回稳定版本实例
            return instances.stream()
                    .filter(instance -> !instance.getMetadata().getOrDefault("version", "v1").equals("v2"))
                    .collect(Collectors.toList());
        });
    }

    private boolean shouldUseGray() {
        // 这里需要从当前请求上下文中获取灰度标记
        // 可以通过 ServerWebExchange 的 attribute 获取
        return false; // 示例
    }
}
然后通过配置类注册这个 Supplier：

java
@Configuration
@ConditionalOnClass(LoadBalancerClientFactory.class)
public class GrayLoadBalancerConfig {

    @Bean
    @Primary
    public ServiceInstanceListSupplier grayServiceInstanceListSupplier(
            LoadBalancerClientFactory clientFactory,
            GrayRuleConfig grayRuleConfig,
            ObjectProvider<LoadBalancerProperties> properties) {
        // 获取默认的 Supplier（可能来自 Nacos）
        ServiceInstanceListSupplier delegate = clientFactory.getLazyInitializingSupplier("default", ServiceInstanceListSupplier.class);
        return new GrayServiceInstanceListSupplier(delegate, grayRuleConfig, properties);
    }
}
这样，负载均衡器在选择实例时，只能看到符合灰度条件的实例列表，无需修改过滤器链。

5. 解决你当前“灰度不生效”的问题

如果你希望保留现有的 GrayRoute 过滤器，确保它能正确覆盖目标地址，请检查：

过滤器的顺序是否 小于 ReactiveLoadBalancerClientFilter 的默认顺序 10150。你可以在 GrayRoute 类上添加 @Order(10000)。
在过滤器中，你必须设置请求的 URL 属性，例如：

java
exchange.getAttributes().put(ServerWebExchangeUtils.GATEWAY_REQUEST_URL_ATTR, uri);
并设置 ServerWebExchangeUtils.GATEWAY_ALREADY_ROUTED_ATTR 为 true，防止负载均衡过滤器再次处理。
确保你从 Nacos 中获取的实例是健康的（instance.isHealthy()），否则负载均衡器可能仍然会尝试选择其他实例。
如果你倾向于用自定义负载均衡器方案，可以完全去掉 GrayRoute 过滤器，将灰度逻辑转移到负载均衡层。

6. 总结

自定义 ReactiveLoadBalancer 或 ServiceInstanceListSupplier 是更符合 Spring Cloud 设计的灰度实现方式，它不侵入过滤器链，且能自动复用负载均衡的容错、重试等特性。你当前遇到的问题——过滤器设置了目标地址但仍路由到非灰度节点——很可能是因为没有正确覆盖负载均衡过滤器的处理。
