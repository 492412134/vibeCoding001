# 基因法+雪花算法变种实现计划

## 需求概述
基于基因法和雪花算法，实现一个变种的雪花算法，用于生成全局唯一的ID。

### 雪花算法结构
- 41比特位-时间戳
- 4比特-政策类型
- 8比特-代表政策分表
- 11比特序列号

### 政策类型
- 0000：common
- 0001：old_man
- 0010：edu
- 0011：live
- 其他做保留位

### 政策分表说明
- 单库有16张表
- 表编号通过政策编号对16取模得到
- 例如政策编号为33010011，表编号为11 (33010011%16=11)

### 序列号说明
- 从Redis中获取序列号
- Redis的key为：政策类型+"_"+政策编号
- 序列号为11位，从0开始递增

## 技术细节

### ID结构（64位）
```
| 时间戳(41位) | 政策类型(4位) | 政策分表(8位) | 序列号(11位) |
| 0-41       | 41-45        | 45-53       | 53-64      |
```

### Redis Key设计
- 序列号Key：`snowflake:seq:{policyType}:{policyCode}`
- 序列号过期时间：1秒（自动清理）

### 并发控制
- 使用Redis的INCR原子操作获取序列号
- 序列号达到2047后从0开始循环
- 同一毫秒内最多生成2048个ID

## 风险与注意事项

1. **时钟回拨问题**：时间戳依赖系统时钟，需要考虑时钟回拨的处理
2. **Redis单点故障**：Redis作为序列号生成器，需要考虑高可用
3. **序列号溢出**：11位序列号最大2047，同一毫秒内超过此数量会冲突
4. **分布式部署**：多节点需要确保Redis连接正常
